# cmd/digest — data inventory and integrity report

**Safety review**: NOT VERIFIED — connects to remote databases (read-only)

## Context

`scripts/safe-data-report.sh` was a prototype that proved the concept: query all data sources and produce a markdown report (`DATA-REPORT.md`). It works but is slow (~5 min), hard to maintain, and can't introspect jsonl files beyond file counts.

`cmd/digest` replaces it in Go, reusing the existing reader infrastructure (mysql, postgres, jsonl packages). It adds cryptographic digests per source for integrity verification.

Inspired by [directory-digester](../../../directory-digester/) which computes hierarchical SHA256 digests for file trees. Here the analog is: entries → day → month → year → source, but using **content-based digests** (not digest-of-digests) so that digests are independent of how data is partitioned.

## Design

### Digest scheme

**Canonical entry format** — identical to jsonl:

```jsonl
{"stamp":"2020-01-01T00:00:00Z","watt":1234}
{"stamp":"2020-01-01T00:00:01Z","watt":1189}
```

Each entry is serialized as one jsonl line (with trailing newline). This means:

- `sha256sum ./data/jsonl/month/2020-01-01T00:00:00Z.jsonl` = the month digest
- Reading the same entries from MySQL or Postgres → same serialization → same digest
- **Existing jsonl files are self-verifying with just `sha256sum`**

**Multi-grain hashing in a single pass:**

```go
dayHash := sha256.New()
monthHash := sha256.New()
yearHash := sha256.New()

// for each entry:
entryBytes := []byte(`{"stamp":"...","watt":...}` + "\n")
dayHash.Write(entryBytes)
monthHash.Write(entryBytes)
yearHash.Write(entryBytes)

// on day boundary: finalize dayHash, start new
// on month boundary: finalize monthHash, start new
// on year boundary: finalize yearHash, start new
```

**Key property**: `digest(month)` = `digest(concat(all day entries))` — the digest is content-based, not structure-based. Whether data lives in 1 month file or 30 day files, the digest is identical because it hashes the same byte stream.

### Source specification

A universal data source descriptor, reusable across commands:

```go
type Source struct {
    Name    string // e.g. "darwin/mysql", "d1-px1/timescaledb", "local/jsonl"
    Type    string // "mysql", "postgres", "jsonl"
    URI     string // connection string or path
    Enabled bool
}
```

URI format per type:

- mysql: `mysql://root@darwin.imetrical.com:3306/ted`
- postgres: `postgresql://postgres:secret@d1-px1.imetrical.com:5432/ted`
- jsonl: `./data/jsonl/`

### Known sources

- `darwin/mysql` — live capture source
- `d1-px1/timescaledb` — live mirror
- `local/jsonl` — `./data/jsonl/`
- `dirac/mysql` — future, disabled until it exists
- `local/timescaledb` — local dev, disabled by default

### Output

**Digest is always present in output.** Additional columns controlled by verbosity flags:

Default output (per source):

| Grain | Period  |    Count | SHA256    |
| ----- | ------- | -------: | --------- |
| month | 2020-01 |  2678400 | abc123... |
| year  | 2020    | 31478859 | bbb222... |

With `--verbose` / flags: add min/max stamp, missing, % missing, min/max watt, zero count:

| Grain | Period | Count | Missing | % Missing | Min Watt | Max Watt | Zeros | SHA256 |
| ----- | ------ | ----: | ------: | --------: | -------: | -------: | ----: | ------ |

Flags:

- `--grain day|month|year` — which levels to include (default: month+year)
- `--verbose` — include all columns (missing, watt range, zeros)

### Progress display (no dependencies)

Live 3-line stderr display using ANSI escape codes (cursor-up + overwrite):

```txt
2020-01-01  3f8a2b...  342M entries
  2020-03-01  a1c9e4...  1.2M entries
    2020-03-15  7b2d01...  52,401 entries
```

- `hash.Sum(nil)` returns intermediate digest without resetting state
- Stderr only — stdout stays clean for piping/redirect
- No charm/bubbletea dependency needed

## Acceptance criteria

- `go run cmd/digest/main.go` produces `DATA-REPORT.md`
- Per-grain digest for each reachable source (always present)
- `sha256sum file.jsonl` = computed month digest for matching data
- Same data across sources → same digest
- Unreachable sources reported gracefully (not fatal)
- watt=0 entries visible when `--verbose`
- Faster than the bash script (~5 min → target <2 min)
